
| **Term**           | **Description**                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Service**        | A self contained unit of functionality designed to perform a specific task (or set of tasks).                                                                                                                                                                                                                                                                                                                          |
| **Client**         | Any piece of hardware or software that is requesting a resource or service from a server.                                                                                                                                                                                                                                                                                                                              |
| **Agent**          | A piece of software that acts on behalf of a user or another program with some form of autonomy. Often it acts as a monitoring service for an environment, and reacts to a change                                                                                                                                                                                                                                      |
| **Daemon**         | A program that runs continually, often in the background. Daemons can be thought of as an agent or a service, depending on functionality                                                                                                                                                                                                                                                                               |
| **Host**           | A host is the physical or virtual machine where a service or an agent exists.                                                                                                                                                                                                                                                                                                                                          |
| **Logs (Logging)** | When you code and use _print_ to debug it, this is a form of “logging”. But this is not possible on larger scales, so instead we “log” to a service, which is just text being written down.<br><br>Logging in it’s purest form is only for remarks on program operation, occasionally it is used colloquially to represent “the collection of data”, this is technically incorrect, however is not the hill to die on. |
| **Proxy**          | A proxy is some service that  acts on behalf of a client, normally to hide the identity of the client. Similarly, a reverse proxy acts on behalf of a service to conceal the services identity.                                                                                                                                                                                                                        |
| **Role**           | A role is part of Role-Based Access Control (RBAC), which defines a set of permissions for what actions (get, put, list, ect ...) can be confirmed by a certain user in a certain namespace                                                                                                                                                                                                                            |
| **Ingress**        | An Ingress is an API that manages external access to internal services the cluster by acting as a router or reverse proxy                                                                                                                                                                                                                                                                                              |
| **Egress**         | Egress refers to outbound network traffic originating from pods inside a cluster destined for external endpoints (like APIs, databases, ect ...)                                                                                                                                                                                                                                                                       |
